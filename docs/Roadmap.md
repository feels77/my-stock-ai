# 🗺️ Roadmap — 생성기1 (my-stock-ai)

> 최종 업데이트: 2026-06-12  
> Product Charter 버전: v1.2 기준  
> 현재 완료 단계: **Phase 1 + Phase R1 + Phase 1.5A + Phase 1.5B**

---

## 🎯 제품 방향 원칙

> "이 앱은 투자 종목을 추천하는 시스템이 아니다.  
> 이 앱은 사용자의 현재 포트폴리오를 건강검진하고,  
> 데이터 기반의 행동 선택지를 제시하는 시스템이다."

**Phase 우선순위 기준**
1. 사용자 데이터 기록 → 분석 → 행동 선택 사이클 완성
2. 실시간 가격보다 **포트폴리오 건강 진단**이 먼저
3. 데이터는 사용자 소유 — 언제든 내보내고 이동 가능

## 🔲 Phase 2 — 황금거위 자산배분 엔진

**목표:** 자산군별 목표 비중 설정 + 현재 비중 비교 + 이상 감지

> Phase 2는 단순 UI 기능이 아니라 **황금거위 자산배분 철학의 핵심 엔진**이다.  
> Phase 3 AI 주간 점검 리포트의 데이터 기반이 되므로 Phase 2 완료 후 Phase 3 착수.

### 자산군 체계 확립

- [ ] `useAssets.js`에 그룹별 집계 함수 추가

## 🔲 Phase 3 — AI 주간 점검 리포트

**목표:** 포트폴리오 현재 상태를 AI가 분석할 수 있는 구조화된 리포트 자동 생성  
**Charter 연결:** "주간 운영 시나리오" 4~5단계 — AI 분석용 리포트 자동 생성

> [!IMPORTANT]
> AI는 종목을 추천하지 않는다. AI는 **포트폴리오 건강 상태를 분석하고 행동 선택지를 제시**한다.  
> 이 Phase의 결과물은 ChatGPT/Claude에 붙여넣는 구조화된 텍스트 리포트다.

### 포트폴리오 요약 리포트 생성
- [ ] "주간 자산 점검" 버튼 → 클릭 1회로 전체 리포트 생성
- [ ] 리포트 구성:
  - 총 평가금액, 총 매입금액, 총 수익률
  - 이번 주 거래 요약 (매수/매도 건수, 거래 금액)
  - 자산 수 / 거래 이력 수

### 자산군 비중 분석 섹션
- [ ] 자산군별 현재 비중 vs 목표 비중 테이블
- [ ] 가장 이탈이 큰 자산군 강조

### 집중 투자 위험 분석 섹션
- [ ] 단일 종목 비중 상위 3개 표시
- [ ] 상위 1개 종목이 전체 30% 초과 시 경고 문구 삽입

### 현금 비중 분석 섹션
- [ ] 현금성 자산(파킹형 + 현금) 비중 계산
- [ ] 현금 비중 10% 미만 시 유동성 부족 경고

### 리밸런싱 제안 섹션
- [ ] 신호등 빨강 자산군에 대해 리밸런싱 필요성 안내 문구 자동 삽입
- [ ] "다음 행동 선택지" 형태로 제시 (결정은 사용자 몫)

## 🔲 Phase 4 — Google Drive 동기화

**목표:** 기기 간 데이터 이동 및 클라우드 백업  
**Charter 연결:** "데이터 소유권 원칙" — 사용자가 언제든 데이터를 내보내고 이동할 수 있어야 함

> [!NOTE]
> Google Drive 연동 전에 **JSON 파일 내보내기/가져오기**를 먼저 구현한다.  
> 파일 백업은 API 없이 구현 가능하고, 데이터 소유권 원칙의 즉각적 실현이 가능하다.

### 선행 구현: JSON 파일 백업/복원
- [ ] "데이터 내보내기" 버튼 → JSON 파일 다운로드
  - `assets`, `transactions`, `targetWeights` 포함
- [ ] "데이터 가져오기" 버튼 → JSON 파일 업로드 후 복원
- [ ] 백업 파일명: `golden_goose_backup_YYYYMMDD.json`

### Google Drive 로그인
- [ ] Google OAuth 2.0 연동
- [ ] 로그인 상태 표시 (로그인/로그아웃 버튼)

### 자동 백업
- [ ] 앱 실행 시 Google Drive에 자동 백업
- [ ] 백업 파일명: `golden_goose_backup_YYYYMMDD.json`

### 복원
- [ ] Google Drive에서 백업 파일 목록 조회
- [ ] 선택한 날짜의 백업 파일 복원
- [ ] 복원 전 현재 데이터 로컬 백업 자동 수행 (안전장치)

## 🔲 Phase 5 — 현재가 자동 업데이트

**목표:** ticker 기반 실시간 주가 조회로 현재가 자동 반영  
**Charter 연결:** "포트폴리오 건강검진"의 정확도 향상 — 현재가 정확해야 비중/손익 계산이 신뢰 가능

> [!NOTE]
> 실시간 가격보다 **포트폴리오 구조 분석(Phase 2, 3)**이 먼저다.  
> 현재가 없이도 수동 입력 현재가로 Phase 2~4까지 완전히 동작할 수 있도록 설계한다.

### 종목 코드 체계 정립
- [ ] 한국 ETF/주식: 6자리 숫자 코드 (예: `360750`)
- [ ] 미국 ETF/주식: 영문 티커 (예: `SPY`, `QQQ`)
- [ ] 구분 로직: 숫자면 한국, 영문이면 미국
- [ ] `ticker` 없는 종목(현금 등)은 업데이트 제외

### 시세 연동 (한국)
- [ ] 한국투자증권 Open API 또는 네이버금융 연동
- [ ] Vite 프록시 설정 (`vite.config.js`)으로 CORS 우회
- [ ] API 오류 시 기존 현재가 유지 (안전 처리)

### 시세 연동 (미국)
- [ ] Yahoo Finance API 또는 대체 무료 API 연동
- [ ] 미국 주식 시장 시간대 처리 (EST 기준)

### 자동 평가금액 계산
- [ ] 전체 업데이트 버튼 → 로딩 상태 + 성공/실패 피드백
- [ ] 업데이트 성공한 종목 수 / 실패한 종목 수 표시
- [ ] 마지막 업데이트 시각 저장 및 표시
