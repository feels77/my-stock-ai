## 1. 제품 정의

### 제품명
**황금거위** (Golden Goose)

### 한 줄 설명
> 개인 자산을 직접 기록·관리하면서, 포트폴리오 건강검진과 AI 투자 코파일럿을 통해 장기 첨부 투자를 스스로 실행하는 개인 자산관리 OS

### 제품 철학
- AI는 도구다. **질문을 잘해야** 좋은 답이 나온다.
- 복잡한 투자 분석을 누구나 쉽게 시작할 수 있어야 한다.
- 내 자산 데이터는 **내 기기에 저장**된다 (localStorage 우선).

---

### 🎯 핵심 정의 (Product Identity)

> **"이 앱은 투자 종목을 추천하는 시스템이 아니다.**
>
> **이 앱은 사용자의 현재 포트폴리오를 건강검진하고,**
> **데이터 기반의 행동 선택지를 제시하는 시스템이다."**

### 핵심 사용 시나리오
1. 매달 매수한 ETF를 앱에 기록한다.
2. 전체 포트폴리오 수익률을 한눈에 확인한다.
3. 관심 종목을 골라 AI 분석 프롬프트를 생성해 ChatGPT/Claude에 붙여넣는다.
4. 아침마다 단타 모닝브리핑 프롬프트를 만들어 퍼플렉시티에서 최신 뉴스를 확인한다.

## 5. 범위 정의 (Scope)

### In Scope (포함)
- 보유 자산 등록/수정/삭제
- 매수·매도 거래 기록 및 자동 반영
- 포트폴리오 수익률 계산 (평가금액, 평가손익, 수익률)
- 현재가 자동 업데이트 (API 연결 후)
- AI 투자 분석 RICE 프롬프트 생성 (10개 카테고리)
- 주식 용어 사전

### Out of Scope (제외)
- 실제 주식 매수/매도 주문 실행 (증권사 연동 없음)
- 세금 계산 (양도소득세, 배당소득세)
- 다중 사용자 / 클라우드 동기화 (현재 단계)
- 소셜 기능 (공유, 팔로우 등)

## 7. 제약 사항 및 원칙

1. **기존 기능 삭제 금지** — 새 기능 추가 시 기존 탭/기능은 유지
2. **데이터 구조 유지** — Asset, Transaction 모델은 하위 호환 유지
3. **UI 변경 최소화** — 리팩터링 시 사용자 화면 동일하게 유지
4. **단계적 개발** — Phase 단위로 검증 후 다음 단계 진행
5. **데이터 소유권** — 사용자의 투자 데이터는 사용자가 소유한다. 모든 데이터는 사용자가 직접 저장·백업·이동할 수 있어야 하며, 특정 서비스에 종속되지 않는 구조를 지향한다. 장기적으로는 Google Drive 등 사용자 계정 기반의 동기화를 지원하되, 사용자가 언제든 자신의 데이터를 내보낼 수 있어야 한다.

## 9. AI 역할 정의 (Portfolio Copilot)

AI는 단순한 질의응답 도구가 아니다.

AI는 사용자의 포트폴리오 상태를 분석하고, 아래 항목에 대한 판단 근거와 행동 선택지를 제공하는 **자산관리 코파일럿** 역할을 수행한다.

| 분석 항목 | 설명 |
|-----------|------|
| 비중 이상 여부 | 특정 자산군에 과도하게 집중되어 있는지 |
| 집중 투자 위험 | 단일 종목/섹터에 대한 위험 노출 수준 |
| 현금 부족 위험 | 유동성 확보 여부 |
| 리밸런싱 필요성 | 목표 비중 대비 현재 비중 괴리 |
| 수익 실현 기회 | 수익률이 높은 자산의 차익 실현 시점 |
| 투자 습관 점검 | 거래 패턴, 매매 주기, 집중도 분석 |

> **원칙:** AI는 사용자의 행동을 대신 결정하지 않는다.  
> AI는 **판단 근거**와 **행동 선택지**를 제공한다.

## 10. 주간 운영 시나리오

Portfolio Doctor의 핵심 사용 패턴은 **주간 점검**이다.

```
1. 사용자는 주중 발생한 거래를 입력한다.
2. 시스템은 현재 자산 상태를 계산한다.
3. 사용자는 "주간 자산 점검" 버튼을 누른다.
4. AI 분석용 리포트가 자동 생성된다.
5. AI는 포트폴리오 상태를 분석한다.
6. 사용자는 이를 참고하여 투자 결정을 내린다.
```

이 사이클이 반복될수록 사용자의 투자 데이터가 축적되고, AI 분석의 정확도가 높아진다.
